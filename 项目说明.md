 时间规划 / 任务管理平台

本地部署 · 前后端分离 · 可扩展的大模型助手 · 多渠道提醒

 项目定位

一个完全本地部署的任务管理与时间规划平台，强调数据隐私、自由定制与轻量级“vibe coding”开发节奏。支持任务看板、日历视图、AI 智能规划、邮件/聊天软件提醒，并可选择接入大学教务系统自动同步课表。

---

技术架构

层级 技术选型建议
后端 Python FastAPI / Django 或 Node.js Express
前端 React / Vue / Svelte + UI 组件库 (Ant Design, Element Plus)
数据库 SQLite（起步）→ PostgreSQL（生产）
AI 助手 Ollama（本地推理，7B 模型） + 可选云端 API（OpenAI 兼容）
定时任务 Celery / APScheduler

大模型助手接入方案

1. 接入方式（混合模式）

· 主推：Ollama 本地部署
  · 模型推荐：deepseek-r1:7b、llama3:8b
  · 硬件需求：显存 ≥4GB（7B 模型），内存 ≥16GB
  · API 地址：http://localhost:11434/api/generate
  · 优势：数据不出本地，延迟低（10–50ms）
· 备选：云端 API
  · 在 Ollama 不可用或需顶级模型时切换到 OpenAI / 其他厂商
  · 代码中预留 base_url 切换逻辑

2. 系统提示词设计（四模块）

```text
# 1. 时间上下文（必须在每次请求动态注入）
今天是 {YYYY-MM-DD}，{星期X}。

# 2. 任务管理准则
- 优先级：🔴高（今天必须完成） > 🟡中（本周完成） > 🟢低
- 任务分解：超过4小时的任务须拆为≤2小时的子任务
- 时间估算：默认每任务45分钟，基于历史数据微调

# 3. 任务拆解规范
- 标题：动词+名词，≤15字
- 预估耗时：精确到15分钟
- 超过2小时的主任务必须分解子任务

# 4. 交互风格
- 称呼用户“你”，以助理口吻对话
- 回复默认 Markdown 格式
- 生成任务后主动询问：“需要我将这些任务添加到待办列表吗？”
```

3. 后端调用示例（FastAPI + Ollama）

```python
@app.post("/ai/chat")
async def chat(prompt: str):
    resp = requests.post("http://localhost:11434/api/generate", 
                         json={"model": "deepseek-r1:7b", "prompt": prompt, "stream": False})
    return {"reply": resp.json()["response"]}
```

多渠道提醒

邮箱提醒（以 QQ 邮箱为例）

· SMTP 配置
  · 服务器：smtp.qq.com，SSL 端口 465
  · 认证：完整邮箱地址 + 16位授权码（在 QQ 邮箱 设置 → 帐户 中开启 SMTP 并生成）
  · 日发送限额：普通用户 ≈100封/天，会员 ≈500封/天
· 发件人说明
    本地部署时，发件人就是你在配置文件里填的那个邮箱。你可以用小号邮箱专门发提醒，不影响收件箱。
· 核心代码（Python）

```python
import smtplib
from email.mime.text import MIMEText

def send_task_reminder(to_email, task_name, due_date):
    msg = MIMEText(f"任务「{task_name}」将在 {due_date} 到期。", "plain", "utf-8")
    msg["Subject"] = f"任务提醒：{task_name}"
    with smtplib.SMTP_SSL("smtp.qq.com", 465) as server:
        server.login("your@qq.com", "your_auth_code")
        server.sendmail("your@qq.com", [to_email], msg.as_string())
```

功能扩展矩阵（按优先级）

第一阶段（MVP+，强烈推荐）

1. 自然语言创建任务
      用户输入“明天下午3点开会”，AI 自动解析标题、截止时间、优先级 → 直接创建任务。
2. 时间日志与统计
      记录每个任务的预估耗时 vs 实际耗时 → 生成周报/月报、效率热力图、任务完成趋势图。
3. 番茄钟 / 专注模式
      内置番茄钟，专注结束后自动记录时长并关联任务，打通“规划→执行”闭环。

第二阶段（数据洞察）

· 效率热力图（日历形式展示每日完成量/专注时长）
· 标签云：统计高频任务标签
· 任务完成率趋势折线图

第三阶段（轻协作）

· 任务委派（局域网内分配给其他用户）
· 共享任务板 + 评论/附件
· 公开只读分享链接（可设密码/有效期）

第四阶段（自动化）

· 简易规则引擎（IFTTT 风格）：当任务优先级变为“高” → 发送邮件提醒
· Webhook 集成：任务变更时通知第三方系统
· 命令行接口（CLI）：task add "买菜" --due "18:00"
· 浏览器插件：快速从网页添加任务

第五阶段（跨设备）

· PWA：可安装到手机桌面，获得离线与推送能力
· 响应式设计：移动端友好操作

接入大学教务系统（获取课表）

可选方案

方案 实现难度 稳定性 推荐度
手工导入 .ics 极低（用户导出→平台导入） 高 ⭐⭐⭐⭐⭐
专用 SDK/API（如 school-sdk、zfu-python） 中等 中（学校系统改版易失效） ⭐⭐⭐
自研爬虫（破解加密、验证码） 极高 低 ⭐

技术难点与风险

· 各高校教务系统（正方、青果等）接口不统一，常有加密、验证码、频率限制。
· 法律与隐私：多数高校禁止外挂工具；要求用户提供明文密码存在极大风险。

安全建议

· 强烈推荐手工导出 .ics，或提供浏览器插件辅助半自动导出。
· 若必须爬虫，应明确提示用户“密码仅存储在本地，我们无法获取”，并建议使用一次性应用密码。
· 获取的课表数据可转换为标准 iCalendar（.ics）格式，便于其他日历软件订阅。

本地部署关键说明

· 发件人是谁：完全由配置文件中的邮箱决定（例如 task_robot@qq.com），程序使用该邮箱登录 SMTP 服务器发送提醒。
· 数据安全：所有数据（含任务、邮件配置、AI 模型）均留在用户服务器/电脑上，无强制云端依赖。
· 启动方式：Docker Compose 一键启动后端 + 前端 + Ollama + 数据库。

4. 增加时间统计和番茄钟
